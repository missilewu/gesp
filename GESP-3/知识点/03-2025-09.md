# GESP三级C++2025年9月考试考点、易错点及相关知识点总结
## 一、考察的知识点（按模块分类）
### （一）基础语法与数据类型
1. 整数除法与类型转换（`int→double`）
2. 逻辑运算符（`&&`、`||`、`!`）的优先级与短路求值
3. 基本数据类型特性（`char`的取值范围、`sizeof`运算符）
4. 变量作用域（局部变量、全局变量、`for`循环内变量的有效范围）
5. 左值与右值，`++i` 和 `i++`的底层逻辑，以及多左值和右值的影响
6. 枚举（`enum`）的类型特性，与整型数据的转换关系
7. 宏定义（`#define`）的语法与副作用
8. 头文件包含（`#include "xxx.h" vs #include<xxx.h>`）的搜索策略
9. `extern`关键字的作用（声明与定义的区别）

### （二）数组与字符串
1. 数组的基本特性：下标规则（从0开始）、声明方式、大小定义（编译时确定）
2. 数组初始化（全局变量初始化是0，局部变量不初始化是内存值，部分初始化时未赋值元素为0）
3. 数组元素访问与边界检查（避免越界）
4. `string`类与字符数组（`char[]`）的区别：赋值（`=` 和 `strcpy`）、长度获取（`length()` 和 `strlen()`）、内存分配、内容比较（`==`的使用）

### （三）函数

1. 函数定义：返回类型、参数类型声明、函数体逻辑（返回两个数的最大值）
2. 函数特性：无参数函数的合法性、`return`语句的使用（`void`函数可含无值`return`）、`main`函数不可被调用（默认被操作系统调用）、函数不能嵌套定义
3. 函数返回值：单个返回值的限制与间接返回多个值的方式
4. 函数参数传递：值传递、引用传递、指针传递（否定“只有值传递”的误区）
5. 形参的生命周期（函数调用结束后释放内存）
6. `C++11`及以后的返回类型要求（必须显式声明，不可默认`int`）

### （四）循环与分支结构
1. `for`循环：初始化、循环条件、步长控制、嵌套循环的执行逻辑
2. `while`循环与`do-while`循环：执行顺序（`do-while`先执行循环体）、`continue`语句的作用（跳过剩余循环体，进入下一次循环）
3. 三元运算符（`?:`）的合法性与左值/右值特性

### （五）编程应用
1. 模拟类问题（数组清零）：遍历数组、查找最值（最大值取下标最大者、最小值排除零）、循环模拟操作流程
2. 日期与日历计算（日历制作）：预计算，月份天数存储、指定月份1号星期几计算（基于已知基准日）、格式化输出（对齐要求、换行控制）

## 二、易错点

### （一）基础语法类
1. 混淆整数除法与浮点除法：认为int a=10, b=3; double c=a/b的结果是3.33333333......。忽略了整数除法先取整再进行转换类型的优先顺序。如果下意识的认为类型转换优先，都转成了double，则也会得出错误的答案
2. 逻辑运算符短路求值错误：如误判`(10!=10)||(5>=3)`的结果，未利用“`||`只要一个为真则整体为真”的短路特性
3. 宏定义的副作用：认为`#define SQUARE(x) x*x`是安全的，未意识到`SQUARE(2+3)`会展开为`2+3*2+3`（结果是11而非25）。记住一个检验的方法，就是把传给宏的表达式，原封不动的替代，展开宏进行验证即可。
4. `char`类型取值范围误区：默认`char`一定是`signed（-128~127）`，忽略`unsigned char（0~255）`的情况
5. 三元运算符赋值合法性：认为`a>b?a:b=10`一定合法，容易下意识的认为优先级是`(a>b?a:b)=10`，实际上=的优先级大于三元运算符，优先级是`a>b?a:(b=10)`。另外需要注意的是，赋值符号（=）的左侧必须是左值（lvalue，有内存地址、可被修改的对象，如普通变量），不能是右值（rvalue，临时值、不可被修改的常量等）。如果a和b的类型不同，编译器会推导出（a>b?a:b）的表达式结果是一个非左值。

### （二）数组与字符串类
1. 数组下标越界：如`int arr[5]`用`arr[5]`访问最后一个元素（实际最后一个元素是`arr[4]`），或循环条件写`i<=SIZE`（SIZE为数组长度）
2. 字符数组比较错误：用`==`比较两个字符数组的内容（实际比较的是地址，应使用strcpy）
3. 数组部分初始化误区：忘记未显式赋值的元素会默认初始化为0（如`int arr[4]={1,2,3}`中`arr[3]`初始为0）

### （三）函数类
1. 函数定义错误：返回类型与return值不匹配（如void函数返回具体值）、参数未声明类型（如`int max(a,b)`）
2. 函数嵌套定义：认为C++允许在一个函数内部定义另一个函数
3. main函数调用误区：试图在其他函数中调用main函数
4. void函数的return使用：认为void函数不能包含return语句（实际可包含无值return，用于提前退出函数）

### （四）循环与流程控制类
1. continue语句作用误解：认为`continue`会终止循环，实际是跳过循环体剩余部分进入下一次循环（如第8题中count++在循环外，执行次数为0）
2. do-while循环执行次数：误以为`do{...}while(false)`不会执行循环体（实际至少执行一次）
3. 嵌套循环输出逻辑混乱：如第9题中未理清外层循环与内层循环的输出顺序，导致误判结果

### （五）编程应用类
1. 数组清零问题：查找最大值时未选择“下标最大的最大值”，或查找最小值时未排除零元素
2. 日历制作问题：计算指定月份1号星期几时基准日错误（已知2025年9月1日是星期一），或格式化输出时未按要求对齐（每个日期占3个字符位置）

## 三、相关的知识点（延伸与拓展）
### （一）基础语法延伸
1. 类型转换：隐式转换（int→double）与显式转换（static_cast）的区别，避免数据丢失
2. 左值与右值：左值可被赋值（如变量、++i），右值不可被赋值（如字面量、i++），C++11中右值引用（&&）的应用
3. 全局变量与静态变量：存储区域（全局/静态存储区）、初始化规则（未显式初始化则为0）、作用域差异（static全局变量仅限当前文件）
4. 头文件包含：""与<>的搜索路径优先级（""先当前目录，再系统目录；<>直接系统目录），避免重复包含（使用#ifndef/#define/#endif）

### （二）数组与字符串延伸
1. 数组边界检查：手动检查下标范围，或使用vector（动态数组）避免越界（GESP三级后续拓展知识点）
2. string类常用成员函数：`assign()`（赋值）、`append()`（追加）、`compare()`（比较）、`substr()`（子串）等
3. 字符数组操作函数：`strcpy()`（赋值）、`strcat()`（拼接）、`strlen()`（长度）、`strcmp()`（比较）的使用注意事项（需包含`<cstring>`）

### （三）函数延伸
1. 参数传递方式：值传递（拷贝实参）、引用传递（&，直接操作实参）、指针传递（*，间接操作实参）的区别与应用场景
2. 函数重载：同一作用域内，函数名相同但参数列表（类型、个数、顺序）不同的函数（GESP三级后续重点）
3. 函数声明与定义分离：头文件中声明函数，源文件中定义函数，避免重复定义
4. 递归函数：函数调用自身（如阶乘、斐波那契数列），需注意终止条件（避免栈溢出）

### （四）循环与流程控制延伸
1. 循环优化：减少循环体内的重复计算（如将数组长度存储到变量，避免每次循环都计算`sizeof(arr)/sizeof(arr[0])`）
2. break与continue的区别：break终止整个循环，continue跳过当前循环剩余部分
3. 循环嵌套的时间复杂度：如双层循环（`O(n²)`），适用于小规模数据（如编程题1中`n≤100`）

### （五）编程应用延伸
1. 模拟类问题通用思路：明确操作步骤→抽象变量→循环模拟→终止条件判断（如数组清零中“全部元素为0”）
2. 日期计算进阶：闰年判断（能被4整除但不能被100整除，或能被400整除）、月份天数动态计算（2月需考虑闰年）
3. 格式化输出：`printf`的格式控制符（`%3d`：占3位右对齐，`%03d`：补零占3位），`cout`的`setw()`和`setfill()`（需包含`<iomanip>`）
